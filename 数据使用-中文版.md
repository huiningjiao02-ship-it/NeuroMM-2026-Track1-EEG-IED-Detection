# 数据使用与伦理说明

## 1. 数据可用性

本仓库不包含 NeuroMM-2026 数据集。该数据集属于受控访问的挑战赛数据，用户需要通过官方渠道申请访问权限，并遵守相应的数据使用协议。

由于该任务涉及临床 EEG 数据，原始数据、标注文件、候选测试数据和模型预测结果均不应被上传到公开仓库中。

## 2. 仓库中刻意排除的文件

以下文件和文件夹不会被上传到公开仓库：

| 文件或文件夹 | 说明 |
|---|---|
| `*.tar`、`*.zip` | 原始压缩数据或提交文件 |
| `*.npy` | EEG 原始或处理后的数组数据 |
| `*.csv` | 标注文件、预测结果或提交文件 |
| `train_data/` | 训练数据 |
| `candidate_data/` | 候选测试数据 |
| `processed/` | 处理后的数据 |
| `annotations/` | 标注文件 |
| `candidate/` | 候选样本 |
| `neuromm26_results/` | 实验结果 |
| `checkpoints/` | 模型权重 |
| `submission.csv` | 提交结果 |
| `sample_submission.zip` | 提交压缩包 |
| `*.pt`、`*.pth`、`*.ckpt` | PyTorch 模型文件或检查点 |

这些文件可能包含原始临床 EEG 数据、衍生数据、标注信息、候选数据、预测结果或训练好的模型参数，因此不应上传至公开仓库。

## 3. 复现实验方式

要复现实验，需要完成以下步骤：

1. 申请官方 NeuroMM-2026 数据集访问权限；
2. 下载 Track 1 所需 EEG 文件和标注文件；
3. 按照 README 中规定的文件名放置数据文件；
4. 在 Google Colab 中运行项目 notebook；
5. 根据 notebook 输出生成结果和提交文件。

## 4. 伦理考虑

临床 EEG 数据需要被谨慎处理。本项目只提供用于学术研究和挑战赛实验的代码，不包含可识别个人身份的信息，也不重新分发原始数据、标注文件、候选数据或衍生数据。

本项目中的模型输出仅适用于研究和挑战赛环境，不应被视为医学诊断结果，也不应直接用于临床决策。

## 5. 公开仓库使用原则

本仓库公开的内容应仅限于：

- 可复现实验代码；
- 环境配置说明；
- 数据路径和运行流程说明；
- 不包含原始数据的实验讨论；
- 不包含个人身份信息、原始 EEG 数据或受控数据的结果总结。

任何包含原始临床数据、标注文件、候选测试数据、模型权重或提交结果的文件，都应保存在本地或受控环境中，不应上传到公开 GitHub 仓库。
